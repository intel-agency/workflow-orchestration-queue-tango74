# AI Repository Summary

## Project: OS-APOW

**Full Name:** Orchestration System — Automated Pipeline Orchestration Workflow

**Repository:** `intel-agency/workflow-orchestration-queue-tango74`

**Description:** A headless agentic orchestration platform that transforms GitHub Issues into autonomous "Execution Orders" fulfilled by specialized AI agents. The system shifts the AI paradigm from an interactive co-pilot to an autonomous background production service.

## Architecture

The system is built on four conceptual pillars:

### 1. Ear (Listener)
- **Technology:** Python 3.12, FastAPI, Uvicorn, Pydantic
- **Responsibility:** High-performance webhook receiver serving as the system's sensory input layer
- **Key Files:** `src/osapow/ear/webhook.py`, `src/osapow/ear/triage.py`
- **Endpoint:** `POST /webhooks/github`

### 2. State (Queue)
- **Technology:** GitHub Issues, Labels, Milestones (no external database)
- **Responsibility:** Task state management using "Markdown as a Database" pattern
- **Key Files:** `src/osapow/state/queue.py`, `src/osapow/state/models.py`
- **Concurrency:** Assign-then-verify distributed locking

### 3. Brain (Orchestrator)
- **Technology:** Python 3.12 (async), Docker CLI, Shell Bridge
- **Responsibility:** Persistent supervisor managing Worker environment lifecycles
- **Key Files:** `src/osapow/brain/orchestrator.py`, `src/osapow/brain/decomposer.py`
- **Features:** Jittered exponential backoff, heartbeat coroutine, graceful shutdown

### 4. Hands (Executor)
- **Technology:** opencode-server CLI, LLM Core, DevContainers
- **Responsibility:** Isolated execution environment where actual coding happens
- **Key Files:** `src/osapow/hands/executor.py`
- **Security:** Network isolation, resource constraints (2 CPU, 4GB RAM)
