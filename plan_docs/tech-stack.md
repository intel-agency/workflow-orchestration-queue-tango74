## 2. Web Framework

| Component | Technology | Purpose |
|-----------|-----------|---------|
| **API Framework** | FastAPI | High-performance async web framework for the Webhook Notifier ("The Ear"). Chosen for native Pydantic integration, automatic OpenAPI generation, and async support. |
| **ASGI Server** | Uvicorn | Production ASGI server to serve the FastAPI application |
| **API Docs** | Swagger / OpenAPI | Auto-generated by FastAPI at `/docs` — provides interactive API documentation for webhook schemas and testing |

## 3. Data & Validation

| Component | Technology | Purpose |
|-----------|-----------|---------|
| **Data Validation** | Pydantic v2 | Strict data validation, settings management, schema definitions for `WorkItem`, `TaskType`, `WorkItemStatus` |
| **Serialization** | Pydantic BaseModel | JSON serialization/deserialization for cross-component communication |

### Data Models (defined in `src/models/work_item.py`)

- **`TaskType`** (Enum): `PLAN`, `IMPLEMENT`, `BUGFIX`
- **`WorkItemStatus`** (Enum): Maps to GitHub Issue labels — `agent:queued`, `agent:in-progress`, `agent:reconciling`, `agent:success`, `agent:error`, `agent:infra-failure`, `agent:stalled-budget`
- **`WorkItem`** (BaseModel): Unified work item with fields: `id`, `issue_number`, `source_url`, `context_body`, `target_repo_slug`, `task_type`, `status`, `node_id`

## 4. HTTP Client

| Component | Technology | Purpose |
|-----------|-----------|---------|
| **HTTP Client** | HTTPX (async) | Fully asynchronous HTTP client for GitHub REST API calls. Used by both Sentinel and Notifier via the shared `GitHubQueue` class. Connection pooling via single `httpx.AsyncClient` instance (per R-5). |

## 6. Containerization

| Component | Technology | Purpose |
|-----------|-----------|---------|
| **Runtime Isolation** | Docker | Worker container sandboxing with network isolation, resource constraints (2 CPUs, 4GB RAM), and ephemeral credentials |
| **Dev Environments** | DevContainers | High-fidelity development environments for AI workers — bit-for-bit identical to human developer environments |
| **Multi-Container** | Docker Compose | Orchestrates complex multi-container needs (e.g., web app + database for integration testing) |

### Container Security

- **Network Isolation:** Worker containers run in a segregated Docker bridge network — cannot access host subnets, metadata endpoints, or peer containers
- **Resource Constraints:** Hard-capped at 2 CPUs and 4GB RAM per worker
- **Ephemeral Credentials:** Tokens injected as temporary env vars, destroyed on container exit, never written to disk

## 10. Logging & Observability

| Component | Technology | Purpose |
|-----------|-----------|---------|
| **Service Logging** | Python `logging` (StreamHandler) | Structured logging to stdout via `StreamHandler`. When running in Docker, stdout is captured by `docker logs`. |
| **Worker Logs** | JSONL files | Raw stdout/stderr captured to local files for forensic analysis (e.g., `worker_run_ID_TIMESTAMP.jsonl`) |
| **Public Telemetry** | GitHub Issue Comments | Sanitized heartbeat comments posted every 5 minutes during task execution. Credential scrubber (`scrub_secrets()`) strips sensitive patterns before posting. |
| **Unique ID** | `SENTINEL_ID` | Every log line stamped with unique sentinel identifier for multi-node tracing |

## 11. Security

| Component | Technology | Purpose |
|-----------|-----------|---------|
| **Webhook Auth** | HMAC SHA-256 | Cryptographic verification of `X-Hub-Signature-256` header on all incoming webhooks |
| **GitHub Auth** | Installation Tokens | GitHub App installation tokens for API access, scoped to least privilege |
| **Credential Scrubbing** | Regex (`scrub_secrets()`) | Strips GitHub PATs (`ghp_*`, `ghs_*`, `gho_*`, `github_pat_*`), Bearer tokens, API keys (`sk-*`), ZhipuAI keys from all public-facing output |
| **Environment Vars** | `os.getenv()` | Secrets via environment variables only — never hardcoded, validated at startup |

### Required Environment Variables

Only 3 required env vars per Simplification Report S-3:

| Variable | Required | Description |
|----------|----------|-------------|
| `GITHUB_TOKEN` | Yes | GitHub App installation token for API access |
| `GITHUB_ORG` | Yes | GitHub organization/owner name |
| `GITHUB_REPO` | Yes | Target repository name (single-repo mode per S-5) |
| `WEBHOOK_SECRET` | Notifier only | HMAC secret for webhook signature verification |
| `SENTINEL_BOT_LOGIN` | Optional | Bot account login for assign-then-verify locking (warns if unset) |
